El SDK oficial `@nekzus/liop` proporciona un conjunto de herramientas completo y fuertemente tipado para construir Clientes LIOP (Agent Nodes) y Servidores LIOP (Data Nodes) en TypeScript y JavaScript.

Si estás migrando desde el Model Context Protocol (MCP), la API te resultará muy familiar. LIOP fue diseñado para ofrecer una experiencia de desarrollo de **reemplazo directo** (drop-in replacement), abstrayendo la complejidad de Protobuf, WebAssembly y la red P2P bajo clases de TypeScript sencillas.

## Arquitectura del Paquete

El SDK se publica en npm como un único paquete que contiene tanto la lógica del Cliente como la del Servidor:

<CodeGroup>
```bash npm
npm install @nekzus/liop@latest zod
```
```bash pnpm
pnpm add @nekzus/liop@latest zod
```
```bash yarn
yarn add @nekzus/liop@latest zod
```
</CodeGroup>

<Note>
  Recomendamos instalar `zod` junto al SDK. Al igual que en la especificación MCP, LIOP utiliza Zod para garantizar que los argumentos de ejecución coincidan con el manifiesto de capacidades del servidor.
</Note>

## Funcionamiento Interno: El Compilador Javy

Si LIOP inyecta lógica compilada en WebAssembly (`.wasm`), ¿cómo es posible desarrollar Agentes puramente en TypeScript?

El SDK incluye un puente de compilación nativo. Cuando un Agente TypeScript invoca `.execute()`, el SDK serializa el payload de JavaScript, lo empaqueta usando un motor similar a Javy en un módulo `.wasm` restrictivo y lo inyecta en la malla a través de gRPC. El desarrollador nunca tiene que escribir ni compilar Rust/Wasm manualmente.

## Características Industriales de Vanguardia

El SDK TypeScript mantiene paridad arquitectónica con el núcleo en Rust, asegurando restricciones Zero-Trust de grado profesional:

- **Criptografía Post-Cuántica (ML-KEM-768):** Encapsulación asimétrica de extremo a extremo (`crystals-kyber`) y autenticación simétrica AES-256-GCM.
- **Transporte de Malla Híbrido:** Conexión P2P sin restricciones para Node.js y navegadores a través de WebSockets (WS) y oyentes libp2p TCP (Yamux).
- **Guardian-TS (Inspección AST en Tiempo Cero):** Mitigación proactiva de escapes del sandbox que bloquea capacidades no autorizadas del host *antes* de que se active la lógica de ejecución `node:wasi`.
- **Listo para ZK-Receipt:** Soporte arquitectónico nativo para Pruebas Matemáticas de Ejecución sin confianza frente a Máquinas Virtuales de Conocimiento Cero (zkVMs).
- **Escalado Multi-Core (Pool de Workers):** Rendimiento transaccional acelerado por hardware usando `piscina`. Supera el cuello de botella del Event Loop de hilo único de V8 al delegar el descifrado PQC, la inspección AST y el Sandboxing WASI a hilos independientes a nivel de sistema operativo.
